使用Hexo+github搭建博客

CSDN广告太多，各种资源还要收费，早就忍受不了了。最近居然动不动就挂掉，只好抽点时间，自己整个blog。
我想要自己编辑MarkDown文件，直接发布到博客。搜索了一下，决定采用Hexo来弄，先借用github存储，后面再转到自己的服务器上。

1. 安装node.js

我安装到/usr/local目录

2. 安装hexo

创建1个目录
安装hexo npm i -g hexo
hexo初始化 hexo init，会自动下载安装

  • node_modules：是依赖包
  • public：存放的是生成的页面
  • scaffolds：命令生成文章等的模板
  • source：用命令创建的各种文章
  • themes：主题
  • _config.yml：整个博客的配置
  • db.json：source解析所得到的
  • package.json：项目所需模块项目的配置信息

3. 关联github

ssh key

ssh-keygen -t rsa -b 4096 -C "user@mail.com" 创建1个新的ssh密钥对
cat ~/.ssh/id_rsa.pub 把里面的内容添加到github的ssh keys

添加repo

github中添加一个username.github.io的目录
修改_config.yml中的部署地址，以后就直接部署到github上

4. 上传到github

先安装 npm install hexo-deployer-git --save（这样才能将你写好的文章部署到github服务器上并让别人浏览到）
执行命令(建议每次都按照如下步骤部署)：

注意deploy的过程中要输入你的username及passward
部署成功后访问 https://username.github.io 就可以看到Hello World页面了

5. 发文章

hexo new “文章名”
在source/_posts会生成后缀名为.md的文件
文件写完后，再执行第4步的命令，发布到github

6. 自建服务器

默认端口为4000, 可修改node_modules/hexo-server/index.js中port字段
启动http服务 hexo server

应该只需要本地和远程同步.md文件就行了，本地编辑好md文件后，同步到远程服务器，服务器上再重新生成静态页面，重启web服务
